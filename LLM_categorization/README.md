Задача: классифицировать массив транзакционных данных покупателей на принадлежность к категории fashion. 

Техническое решение:
- Тестирование LLM (DeepSeek) для автоматической классификации товаров

Основные этапы:
- Сравнение качества разметки (Ручная разметка vs Автоматическая разметка с помощью DeepSeek API)
- Оптимизация процесса классификации:
- Разработка асинхронной системы запросов к API
- Ускорение обработки больших объемов данных
- Обработка ошибок и повторные попытки при сбоях


Ключевые результаты

Качество классификации:
- F1-score: 0.96
- Precision: 0.96
- Recall: 0.96
- Accuracy: 0.98

Другие параметры:
- Асинхронная архитектура с контролем параллелизма (до 8 одновременных запросов)
- Оптимизация скорости: обработка 1000 записей за ~4 минуты (4.38 запросов/сек)
- Стабильность: реализована система повторных попыток и обработки ошибок
- Баланс между скоростью и надежностью через настройки таймаутов и повторных попыток
